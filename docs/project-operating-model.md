# Project Operating Model

## Engagement types

### 1. Quick landscape scan

Best for early client conversations.

- 10–15 queries
- 200–400 raw rows
- 5–10 clusters
- 1 client memo
- 3–5 positioning hypotheses

### 2. Strategy sprint

Best for clients choosing a content or offer direction.

- 25–40 queries
- 750–1,500 raw rows
- cluster scoring
- competitor channel review
- Google Trends comparison
- client workshop

### 3. Repeatable monitoring

Best for clients tracking a niche over time.

- fixed query set
- monthly or quarterly collection
- trend tracking
- new channel detection
- topic freshness review

## Roles

### Research lead

Owns method, quality, and synthesis.

### Data collector

Captures search results, metadata, and notes.

### Analyst

Clusters videos, scores opportunities, and identifies gaps.

### Strategist

Turns findings into client positioning, content ideas, and offers.

## Deliverables

- query map
- research spreadsheet
- cluster scorecard
- client memo
- content wedge recommendations
- opportunity backlog
- optional slide deck

## Quality bar

A useful result should be:

- specific enough to act on
- honest about uncertainty
- grounded in observable examples
- clear about what is crowded vs underserved
- connected to the client’s strengths

## Anti-patterns

Avoid:

- claiming exact YouTube coverage counts
- using only one search phrase
- judging content only by view count
- ignoring audience language
- presenting a spreadsheet without synthesis
- recommending generic “make better content”
